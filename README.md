Спрайты можно было поместить в Asset Bundle для более удобного использования при расширении игры. Также можно было применить Adressables и вынести бандлы на сервер для подгрузки необходимых ресурсов на лету по мере необходимости в игре

В реальном большом проекте, когда много ресурсов используется в игре, можно инстанцировать префабы по мере использования или на этапе инициализации, и удалять со сцены, когда они не нужны

Вкладки книги инструментов можно было поместить в Horizontal Layout Group опять же для более удобного добавления новых вкладок при расширении игры и для более удобного управления размещением элементов вкадок

Сейчас эти действия не были сделаны для более быстрого прототипирования, для более быстрого результата

Для анимаций используется DOTWeen

Нарезки спрайта руки не было в ресурсах, сделал без руки, но с анимациями взятия, применения и возврата инструмента

У спрайта чистого лица жёлтые тени у глаз, потому что я нарезал из заготовок ресурсов, а там такое было чистое лицо. Технически, все смены спрайтов макияжа в игре происходят правильно.
___________________________________

Кончик кисточки не красил, поскольку не было нарезки для этого. А так, сделать это было бы просто. Я бы на инструмент кисти добавил бы спрайты краски, и включал бы плавно аналогично тому, как делал это для теней и румян
___________________________________

У книги инструментов есть 3 вкладки: с тенями, с пудрой/румянами, и с помадой, и 4-ая вкладка пустая.
С вкладками сделал, гляда на макет в ресурсах к игре

Для инструментов, перемещаемых рукой, используется базовый класс Tool. От него наследуется класс Brush, который имеет базовую функциональность для кисти теней и кисти румян. То есть от класса Brush наследюутся классы EyeBrush и BlushBrush. И также от базового класса Tool наследуется класс помады Lipstick
___________________________________

Можно было применить подход, при котором вместо использования классов MonoBehaviour и прокидывания ссылок в сериализованные поля используются обычные классы, в которые прокидываются transform соответствующих объектов на сцене, а необходимые компоненты находятся на этапе инициализации через GetComponent() и Find().
